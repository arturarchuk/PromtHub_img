# Чеклист качества промпта

Перед тем как выдать prompt, проверь его по этому списку.

## Уровень prompt

- Выбран ли правильный уровень из `Уровни-промпта.md`?
- Не стал ли простой prompt слишком длинным?
- Достаточно ли жесткий lock prompt для лица, товара, серии или референсов?
- Если это local edit, ограничена ли область правки?

## Главный объект

- Понятно ли, кто или что главное?
- Не конкурируют ли несколько объектов за внимание?
- Есть ли описание внешности, формы, материала или важной детали?

## Сцена

- Понятно ли, где происходит сцена?
- Есть ли фон или окружение?
- Есть ли время суток, если оно важно?

## Композиция

- Указан ли план: close-up, medium shot, full body, wide shot, overhead, low angle?
- Есть ли понятное расположение объекта?
- Не перегружен ли кадр?

## Свет и цвет

- Указан ли источник света?
- Понятна ли цветовая палитра?
- Свет подходит задаче?

## Материалы и фактуры

- Есть ли видимые текстуры?
- Для товара: материал, поверхность, отражения.
- Для человека: ткань, кожа, волосы.
- Для еды: пар, глянец, свежесть, структура.

## Стиль

- Стиль выбран под задачу?
- Нет ли случайных имен режиссеров, художников, брендов или film stock?
- Не стал ли prompt слишком "кинематографичным" там, где нужна схема, товар или постер?

## Ограничения

Добавлены ли нужные запреты:

- no text / no watermark
- no distorted anatomy
- no extra fingers
- preserve identity
- do not distort product shape
- clean readable typography

## Финальная проверка

- Можно ли сразу скопировать English prompt в Nano Banana 2?
- Не потеряна ли идея пользователя?
- Не слишком ли prompt длинный для простой задачи?
- Есть ли сначала русский prompt, потом английский prompt в markdown code block?
- Следующий этап направляет в веб-Gemini и просит результат + обратную связь?

## Проверка уточнений

Перед тем как писать финальный prompt, спроси себя:

- есть ли критичная неизвестная деталь?
- есть ли противоречие?
- понятна ли роль референсов?
- понятен ли точный текст, если он нужен?
- понятно ли, что сохранить при editing?

Если ответ "нет" по критичному пункту - задай 1-3 уточняющих вопроса вместо финального prompt.

## Проверка editing

- явно написано, что изменить?
- явно написано, что оставить таким же?
- сохранены перспектива, свет, масштаб и тени?
- нет лишнего полного переописания, которое может поменять всю картинку?

## Проверка consistency

- для персонажа указаны identity/face locks, если нужно?
- для товара указаны shape/logo/product locks, если нужно?
- для серии указаны постоянные якоря: персонаж, стиль, палитра, локация?

## Проверка reference priority

- у каждого референса есть роль?
- есть ли priority order, если референсов несколько?
- запрещено ли брать лицо из outfit/style reference?
- product reference защищен от redesign?
- указано ли, что игнорировать из нецелевых референсов?

## Проверка анти-паттернов

- prompt не состоит только из красивых абстракций?
- позитивное описание сильнее, чем список запретов?
- нет лишней cinematic-стилизации для товара, схемы или постера?
- для рук есть hand choreography или руки логично скрыты?
- для точного текста есть кавычки, readable typography и запрет лишних букв?

## Проверка после результата

- если пользователь прислал только картинку, запрошена ли `Feedback-Form.md`?
- есть ли оценка результата по `Quality-Score-Rubric.md`, если нужен разбор?
- если есть v1/v2/v3, ведется ли логика `Prompt-Versioning.md`?
- если пользователь дал feedback, отделены ли desired changes от actual errors?
