# Как работает PromptHub

PromptHub - это локальная система для написания промптов под Nano Banana 2. Основной сценарий: PromptHub пишет prompt, пользователь генерирует в веб-Gemini по своей подписке, затем присылает результат и свою обратную связь для разбора и v2/v3. Локальный API остается дополнительным режимом для явных API-задач.

## Главный сценарий

Пользователь пишет обычным языком:

"напиши промт: девушка в дорогом ресторане ночью"

Если запрос понятный, проект сразу выдает:

Русский prompt:
...

English prompt:
```text
...
```

## Если запрос непонятный

Если запрос слишком общий, противоречивый или не хватает критичных данных, проект сначала задает 1-3 вопроса.

По рабочему правилу проект обычно задает 1-3 коротких уточняющих вопроса перед финальным prompt даже при понятной идее, чтобы лучше попасть во вкус пользователя. Если пользователь пишет "реши сам", "на твой вкус", "сразу prompt" или "без вопросов", проект не уточняет и сразу собирает prompt.

Пример:

"сделай рекламную картинку"

Проект должен спросить:

1. Что рекламируем?
2. Для какой платформы нужен формат?
3. Нужен ли текст на изображении?

После ответа пользователя проект собирает финальный prompt.

## Основной pipeline

1. Определить тип задачи:
   generation / editing / reference composition / refinement / critique / save memory.

2. Проверить, хватает ли данных.

3. Обычно задать 1-3 коротких уточняющих вопроса перед финальным prompt.

4. Если пользователь просит "реши сам", "на твой вкус", "сразу prompt" или "без вопросов" - выбрать defaults и сразу написать prompt.

5. Собрать production-ready prompt.

6. Направить пользователя в web-Gemini: сгенерировать по English prompt, прислать результат и короткую обратную связь.

7. В конце ответа предложить следующий этап.

8. После результата принять обратную связь и сделать v2/v3.

## Главное правило UX

Пользователь не должен говорить специальными командами.

Можно писать:

"норм, но лицо пластиковое"
"не тот вайб"
"свет плоский"
"сделай дороже"

Проект должен сам понять, что это обратная связь, записать короткий урок и дать v2.

## Следующий этап

В конце каждого завершенного ответа проект всегда дает одно короткое предложение следующего этапа: что проверить, протестировать, уточнить или улучшить дальше.
