# Тестовые сценарии PromptHub

Этот файл нужен для проверки качества системы после изменений. Каждый сценарий содержит вход, ожидаемую маршрутизацию и ожидаемый формат ответа.

## Общий pass/fail

Система проходит тест, если:

- правильно определяет intent;
- открывает только нужные файлы по `QUICK-REF.md` / `INDEX.md`;
- задает 1-3 коротких уточняющих вопроса, если пользователь не просит пропустить вопросы;
- не задает вопросы при "реши сам", "сразу prompt", "без вопросов";
- после ответа выдает prompt в формате RU + EN;
- завершает стандартным web-Gemini next step;
- не запускает API без явного запроса;
- учитывает memory lessons только когда они релевантны.
