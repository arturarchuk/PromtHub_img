# Тестовые сценарии PromptHub

Этот файл нужен для проверки качества системы после изменений.

## Как тестировать

Для каждого сценария система должна:

1. Правильно определить intent.
2. Подключить только нужные файлы.
3. Задать 1-3 коротких уточняющих вопроса перед финальным prompt, если пользователь не просит пропустить вопросы.
4. После ответа пользователя выдать prompt в формате RU + EN.
5. Завершить ответ web-Gemini next step: сгенерировать по English prompt и прислать результат + обратную связь.
6. Не запускать API, если пользователь явно не попросил API-запуск.
7. Не перегрузить prompt лишними деталями.

## Pass/fail для web-Gemini workflow

Система проходит тест, если:

- для обычного prompt направляет пользователя в веб-Gemini;
- перед финальным prompt задает немного вопросов комфортно: обычно 1-2, максимум 3;
- не задает вопросы, если пользователь просит "реши сам", "сразу prompt" или "без вопросов";
- просит после генерации прислать не только изображение, но и feedback;
- не делает v2 только по изображению без пользовательского направления, если проблема не очевидна;
- сохраняет стандартный формат RU + EN;
- учитывает memory lessons только когда они релевантны.

## Сценарии prompt generation

### 1. Короткий портрет

Запрос:
девушка в дорогом ресторане ночью

Ожидание:
Сначала 1-2 вопроса про стиль/формат или главный акцент. После ответа - prompt. Портретный шаблон, 4:5, теплый ресторанный свет, readable exposure, natural skin. Следующий этап просит web-Gemini результат + feedback.

### 2. Product photography

Запрос:
черные мужские часы на черном мраморе, дорого

Ожидание:
Сначала 1-2 вопроса про формат/фон/назначение. После ответа - prompt. Товарный шаблон, 1:1 или 4:5, controlled reflections, no warped product shape. Следующий этап просит проверить форму, материал, фон.

### 3. Beauty editorial

Запрос:
премиальный beauty портрет девушки с кремом, дорого и реалистично

Ожидание:
Использовать `Шаблон-beauty-editorial.md`, realistic skin micro-texture, sculpted editorial lighting, no plastic retouching, cream as secondary accent.

### 4. Food photography

Запрос:
аппетитная реклама клубничного десерта в ресторане

Ожидание:
Использовать `Шаблон-food.md`, текстуры еды, свет, сервировка, no plastic-looking food.

### 5. Интерьер

Запрос:
дорогой интерьер ресторана вечером, теплый свет

Ожидание:
Использовать `Шаблон-интерьер-архитектура.md`, materials, architectural lighting, straight vertical lines, realistic scale.

### 6. Постер с неизвестным текстом

Запрос:
сделай постер для вечеринки с надписью

Ожидание:
Уточнить точный текст и формат. Не писать финальный prompt до ответа.

### 7. Постер с точным текстом

Запрос:
постер для techno вечеринки, текст "NIGHT SIGNAL", 2:3

Ожидание:
Задать 1-2 коротких вопроса про визуальный стиль или атмосферу. Exact text, typography hierarchy, no extra letters.

### 8. Инфографика

Запрос:
инфографика про 5 этапов создания промпта

Ожидание:
Использовать `Шаблон-инфографика.md`, exact labels if known, clean layout, readable text, 16:9 или 4:5.

## Сценарии references / editing

### 9. Редактирование фото

Запрос:
на фото поменяй фон на дорогой ресторан, лицо и позу оставь

Ожидание:
Editing prompt. Preserve face, pose, body proportions, lighting consistency, change only background.

### 10. Два референса с конфликтом

Запрос:
Image A лицо девушки, Image B платье на другой модели. Сделай ее в этом платье.

Ожидание:
Use Reference-Priority-Protocol. Image A strict identity, Image B outfit only, ignore face from Image B.

### 11. Серия изображений

Запрос:
сделай 4 кадра одной героини в одном стиле

Ожидание:
Использовать `Шаблон-storyboard-серия.md`, character lock, outfit lock, palette, camera language, per-frame action.

### 12. Product consistency

Запрос:
сделай 3 рекламных варианта одного флакона, форму и логотип сохранить

Ожидание:
Использовать `Шаблон-character-product-consistency.md`, product lock, no redesign, no warped product shape.

## Сценарии feedback

### 13. Результат без feedback

Запрос:
[пользователь прислал только изображение]

Ожидание:
Не делать v2 сразу, если проблема не очевидна. Попросить форму: что нравится, что не нравится, что сохранить, что изменить, оценка 1-10.

### 14. V2: пластиковое лицо

Запрос:
норм, но лицо пластиковое и глаза неживые

Ожидание:
Записать урок с тегами #face #lighting. Выдать v2 с realistic skin micro-texture, natural catchlights, no plastic retouching.

### 15. V2: руки странные

Запрос:
в целом ок, только руки странные, лицо и свет сохранить

Ожидание:
Записать урок. Preserve face and lighting. Если руки не важны, спрятать; если важны, задать hand choreography.

### 16. Clean restart

Запрос:
уже третья версия, стало хуже, лицо поплыло и стиль грязный

Ожидание:
Использовать `Prompt-Versioning.md`, предложить clean restart: взять лучший результат как reference, сохранить рабочие элементы, убрать конфликтующие инструкции.

## Сценарии API

### 17. API-запуск

Запрос:
запусти генерацию через API

Ожидание:
Использовать API workflow только потому, что пользователь явно попросил. Не показывать ключ. Если quota/billing ошибка, объяснить, что API billing отдельно от веб-подписки.
