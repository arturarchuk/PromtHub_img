# INDEX — Дерево решений PromptHub

> Не читай все файлы подряд. Найди свой тип задачи ниже → открой только нужные файлы.

---

## Шаг 1 — Какой у тебя тип задачи?

```
Что нужно сделать?
│
├── 🆕 Создать новое изображение с нуля ──────────────── → [A] Новая генерация
│
├── ✏️  Изменить существующее фото ───────────────────── → [B] Редактирование
│
├── 🔁 Улучшить промпт по фидбеку (v2/v3) ────────────── → [C] Итерация
│
├── 🎭 Серия / кампания / несколько кадров ────────────── → [D] Серия
│
├── 📋 Оценить или разобрать готовый промпт ───────────── → [E] Критика
│
└── 💾 Сохранить результат / урок в память ────────────── → [F] Память
```

---

## [A] Новая генерация

### A1 — Выбери категорию

```
Что на изображении?
│
├── 👤 Человек / портрет / fashion
│   ├── Без конкретного лица          → Шаблон-портрет.md  [Level 2]
│   ├── С референсом лица             → Шаблон-портрет.md  [Level 4]
│   │                                   + Консистентность-лица-Nano-Banana.md
│   └── Beauty / skincare / editorial → Шаблон-beauty-editorial.md  [Level 2–3]
│
├── 📦 Товар / продукт / упаковка
│   ├── Без бренда                    → Шаблон-товар.md  [Level 2]
│   └── С логотипом / брендом         → Шаблон-товар.md  [Level 4]
│                                       + Reference-Priority-Protocol.md
│
├── 🎬 Постер / баннер / thumbnail    → Шаблон-постер.md  [Level 2–3]
│
├── 🌆 Сцена / кинематографика        → Шаблон-сцена.md  [Level 2–3]
│
├── 🍕 Еда / напитки                  → Шаблон-food.md  [Level 2]
│
├── 🏛️  Интерьер / архитектура         → Шаблон-интерьер-архитектура.md  [Level 2]
│
├── 📊 Инфографика / схема / диаграмма → Шаблон-инфографика.md  [Level 3]
│
└── 🎭 Персонаж / маскот / бренд-герой → Шаблон-character-product-consistency.md  [Level 4]
```

### A2 — Всегда перед выдачей

1. Проверь `Анти-паттерны-и-ошибки.md`
2. Сверься с `Чеклист-качества-промпта.md`
3. Если нужна помощь с уточняющими вопросами → `Уточняющие-вопросы.md`

## [B] Редактирование фото

```
Шаблон-редактирование-фото.md  [Level 5]
│
├── Меняем фон / среду           → укажи что сохранить: лицо, поза, пропорции
├── Меняем одежду / стиль        → зафиксируй identity явно
├── Меняем освещение             → сохрани композицию и субъект
└── Убираем / добавляем элемент  → Level 5: Edit only [X]. Keep everything else.
```

Дополнительно: `Reference-Priority-Protocol.md` если используется референс

---

## [C] Итерация v2/v3

```
Пользователь дал обратную связь
│
├── Протокол-улучшения-v2-v3.md   ← главный файл
├── Правила-обратной-связи.md     ← как интерпретировать фидбек
└── Memory-Tags.md                ← какие теги добавить в журнал
```

**Правило деградации:** если 3–5 итераций ухудшают результат → консолидированный перезапуск от лучшей версии

## [F] Сохранение в память

```
Удачный результат   → Мои-удачные-промты.md
Ошибка / урок       → Журнал-экспериментов.md  (формат: сцена/проблема/причина/правило/теги)
Теги для записи     → Memory-Tags.md
```

## Быстрый справочник: уровни промпта

| Level | Применение |
|-------|-----------|
| 1 — Quick | Простая задача, низкий риск |
| 2 — Production | Коммерческий, fashion, product, poster |
| 3 — Structured | Сложная сцена, текст в кадре, несколько объектов |
| 4 — High-Risk Lock | Конкретный человек, продукт с брендом, серия |
| 5 — Local Edit | Точечная правка одного элемента |

Детали → `Уровни-промпта.md`
