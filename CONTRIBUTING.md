# CONTRIBUTING — Как вносить вклад в PromptHub

Этот документ описывает правила командной работы с репозиторием. Читать при первом знакомстве и при любых сомнениях.

---

## Базовые правила

- Все работают в ветке **main** напрямую
- **Перед записью в любой файл — читай его текущую версию** (другие участники могли внести изменения)
- Улучшай и дополняй содержимое, не ломая существующую логику и структуру
- При значимом изменении правил — добавь запись в `CHANGELOG.md`

---

## Как добавить запись в журнал экспериментов

Файл: `Журнал-экспериментов.md`

**Формат записи:**
```
## [ГГГГ-ММ-ДД] Краткое название кейса
**Теги:** #tag1 #tag2 #tag3
**Сцена:** [что снимали, контекст]
**Проблема:** [что пошло не так]
**Причина:** [почему это произошло]
**Правило:** [вывод для будущего применения]
**Рейтинг лучшей итерации:** X/10
```

**Правила:**
- Добавляй 2–5 тегов из `Memory-Tags.md` — не придумывай новые без необходимости
- Записывай только проверенные выводы, не гипотезы
- Одна запись = один конкретный урок

---

## Как добавить удачный промпт

Файл: `Мои-удачные-промты.md`

**Критерии для добавления:** рейтинг 8/10 и выше, результат подтверждён реальной генерацией

**Формат:**
```
## [Тип задачи] Название / краткое описание (Рейтинг: X/10)
**Дата:** ГГГГ-ММ-ДД
**Теги:** #tag1 #tag2

**Русский prompt:**
[промпт]

**English prompt:**
[промпт]

**Ключевой принцип:** [одно предложение о том, что сделало промпт успешным]
```

## Как обновлять шаблоны

Файлы: `Шаблон-*.md`

**Разрешено:**
- Добавлять новые примеры в раздел `## Пример`
- Уточнять формулировки существующих правил
- Добавлять новые constraints на основе реального опыта

**Не разрешено без обсуждения:**
- Менять базовую структуру шаблона
- Удалять существующие разделы
- Изменять формат prompt-вывода

---

## Как обновлять AGENTS.md

`AGENTS.md` — единственный источник правил для агента. Изменения в нём влияют на всё поведение системы.

**Перед изменением:**
1. Убедись, что правило проверено на реальных запросах
2. Не дублируй правила из других файлов — ссылайся на них
3. После изменения добавь запись в `CHANGELOG.md`

---

## Соглашение об именовании файлов

| Тип | Формат | Пример |
|-----|--------|--------|
| Шаблоны | `Шаблон-[категория].md` | `Шаблон-портрет.md` |
| Протоколы | `[Тема]-Protocol.md` | `Reference-Consistency-Protocol.md` |
| Гайды | `[Тема]-Guide.md` | `API-Guide.md` |
| Рабочие процессы | `[Инструмент]-Workflow.md` | `Gemini-Web-Workflow.md` |
| Быстрые справочники | `[TOPIC]-REF.md` | `QUICK-REF.md` |

**Для русскоязычных файлов:** слова через дефис, без пробелов, первое слово с заглавной.

---

## Как обновлять Gallery.md

Файл: `Gallery.md`

Добавляй строку в таблицу после каждой успешной генерации с рейтингом 7/10 и выше:

```
| ГГГГ-ММ-ДД | Тип задачи | Краткое описание промпта | X/10 | #теги | Заметка |
```

---

## Что делать с устаревшим контентом

- **Устаревшие правила** — удаляй или заменяй, добавь запись в CHANGELOG
- **Дублирующие файлы** — объединяй, старый файл удаляй
- **Экспериментальные заметки** — переносить в журнал, если есть вывод; удалять, если нет

---

## Чего не делать

- Не создавай файлы с одинаковым смыслом (проверяй INDEX.md перед созданием нового файла)
- Не добавляй правила, не проверенные практикой
- Не записывай API-ключи в любые файлы репозитория
- Не переименовывай файлы без обновления всех ссылок на них в INDEX.md и AGENTS.md
