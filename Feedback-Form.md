# Feedback Form

Этот файл задает короткую форму обратной связи после генерации в веб-Gemini.

## Когда использовать

Используй после того, как пользователь прислал результат генерации.

Не делай v2 только по изображению, если пользователь еще не сказал, что именно он хочет видеть. Сначала попроси короткую обратную связь по этой форме.

## Минимальная форма

Попроси пользователя ответить коротко:

1. Что нравится?
2. Что не нравится?
3. Что обязательно сохранить?
4. Что изменить в v2?
5. Насколько близко к цели: 1-10?

## Расширенная форма

Используй для сложных задач, референсов, лиц, товаров, постеров и editing.

Идея / цель:
[что хотели получить]

Что получилось хорошо:
[1-3 пункта]

Что не получилось:
[1-3 пункта]

Что сохранить:
[лицо / позу / товар / фон / свет / стиль / текст / композицию]

Что изменить:
[конкретные изменения для v2]

Критичные ошибки:
[лицо не похоже / руки / текст / товар / свет / фон / стиль]

Оценка близости:
[1-10]

Следующее действие:
v2 / v3 / clean restart / сохранить как удачный prompt

## Как использовать обратную связь

1. Отдели мнение пользователя от собственной диагностики.
2. Не записывай желаемое изменение как ошибку.
3. Если пользователь хочет сохранить элемент, явно зафиксируй его в v2.
4. Если пользователь указал ошибку, добавь позитивный фикс и короткий constraint.
5. Если оценка ниже 5/10, чаще нужен clean restart.
6. Если оценка 6-8/10, чаще нужна точечная v2.
7. Если оценка 9-10/10, сохраняй удачную формулу в `Мои-удачные-промты.md`.

## Быстрый ответ пользователю

Если пользователь прислал только картинку без комментария:

Пришли коротко:
1. что нравится,
2. что не нравится,
3. что сохранить,
4. что изменить в v2,
5. оценку близости 1-10.

После этого я сделаю точный v2.
