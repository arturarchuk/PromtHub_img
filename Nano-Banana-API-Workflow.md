# Nano Banana API Workflow

Этот файл описывает безопасное подключение PromptHub к Google AI Studio / Gemini API.

## Важное правило безопасности

Никогда не вставляй реальный API key в:

- чат
- Markdown-документы
- скриншоты
- код проекта
- git commit

Если ключ был отправлен в чат или показан публично, считай его раскрытым:

1. Удали или отзови старый ключ в Google AI Studio.
2. Создай новый ключ.
3. Храни его только локально в `.env` или в системной переменной окружения.

## Роль PromptHub

PromptHub остается prompt-writing системой по умолчанию, но может запускать локальную генерацию по явной команде пользователя.

Проект:

- пишет production-ready prompts
- хранит версии prompt
- фиксирует ошибки и удачные формулы
- помогает готовить API workflow
- помогает анализировать результаты после генерации
- запускает локальный API runner, если пользователь явно попросил генерацию или тест API

Проект не генерирует изображения в обычном prompt-writing сценарии. Генерация разрешена только как явный API-запуск внутри проекта.

## Локальная настройка

1. Создай новый ключ в Google AI Studio.
2. Открой уже созданный `.env`.
3. Вставь новый ключ вместо placeholder:

```bash
GOOGLE_API_KEY=PASTE_YOUR_NEW_SECRET_HERE
GOOGLE_AI_PROJECT_ID=projects/168169110837
```

4. Не добавляй `.env` в git. Он уже закрыт через `.gitignore`.
5. Запусти:

```bash
scripts/run_nano_banana_test.sh
```

## Проверка доступа без сохранения ключа

Можно временно задать ключ в терминале:

```bash
export GOOGLE_API_KEY="your_new_key_here"
```

Не пиши реальный ключ в команду, которую нужно показывать кому-то еще.

## Локальный тест

Для проверки API в проекте есть:

- `API-Test-Quickstart.md`
- `scripts/nano_banana_api_test.py`

Скрипт берет ключ из `.env` или переменной окружения и сохраняет тестовые изображения в `outputs/`.

## Запуск генерации ассистентом

Ассистент может запускать генерацию, если пользователь явно попросил:

- "запускай генерацию"
- "тестим API"
- "сгенерируй через API"
- "создай output"

Запускать можно только локальные скрипты проекта. В ответе нельзя показывать API key.

## Что проверять в Google AI Studio

- API key создан на правильном аккаунте.
- Project ID совпадает с нужным проектом.
- В разделе Billing нет ошибок.
- В разделе Rate Limit видны доступные лимиты.
- В документации или playground доступны image generation модели.

## Названия моделей

В интерфейсе Gemini может быть название "Nano Banana 2", но в API обычно нужен точный model id.

Официальные image generation model ids:

- `gemini-2.5-flash-image` - Nano Banana / быстрый image model.
- `gemini-3-pro-image-preview` - Nano Banana Pro / Pro image model.

Если нужен Pro-тест, поставь в `.env`:

```bash
NANO_BANANA_MODEL=gemini-3-pro-image-preview
```

## Возможные ошибки

### API key not valid

Причина:
ключ удален, скопирован не полностью или создан в другом аккаунте.

Фикс:
создать новый ключ и обновить `.env`.

### Billing required

Причина:
подписка Gemini Pro не всегда равна API billing.

Фикс:
открыть Billing в Google AI Studio / Google Cloud и привязать оплату к проекту.

### Model not found

Причина:
модель недоступна в регионе, аккаунте, API версии или проекте.

Фикс:
проверить актуальные названия image generation моделей в Google AI Studio / official Gemini API docs.

### Quota exceeded

Причина:
закончился free tier или лимит подписки/API.

Фикс:
проверить Usage, Rate Limit и Billing.

## Рекомендуемый рабочий цикл

API workflow не является основным путем. Основной путь описан в `Gemini-Web-Workflow.md`.

Если пользователь явно выбрал API:

1. PromptHub пишет prompt в стандартном формате RU + EN.
2. Локальный API runner отправляет English prompt.
3. Результат сохраняется локально в `outputs/`.
4. Пользователь смотрит результат и дает обратную связь: что нравится, что не нравится, что сохранить, что изменить.
5. PromptHub анализирует результат через обратную связь пользователя.
6. Если есть проблемы, PromptHub записывает урок в `Журнал-экспериментов.md` и делает v2/v3.

## Минимальный формат результата для анализа

Когда присылаешь результат, полезно указать:

- какой prompt использовался
- какая модель использовалась
- что получилось хорошо
- что не понравилось
- что нужно сохранить
- что нужно изменить
- оценку близости 1-10
- нужно ли сохранять лицо, товар, стиль или композицию

Если есть изображение результата, можно прислать его вместе с короткой обратной связью.
