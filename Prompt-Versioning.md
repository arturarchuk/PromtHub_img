# Prompt Versioning

Этот файл нужен, чтобы вести версии prompt внутри одной задачи.

## Зачем

Без версий легко потерять, что именно менялось между v1, v2 и v3. Версионирование помогает не наращивать хаос и вовремя делать clean restart.

## Формат версии

Используй короткую запись:

---

Проект:
[короткое название]

Версия:
v1 / v2 / v3 / restart-v1

Дата:
YYYY-MM-DD

Теги:
[#tag #tag]

Цель:
[что хотели получить]

Prompt:
[коротко или ссылка/название prompt, полный prompt только если нужно]

Что изменилось относительно прошлой версии:
[1-3 пункта]

Результат:
[что получилось]

Обратная связь пользователя:
[что нравится / не нравится / сохранить / изменить]

Решение:
continue / local edit / v2 / v3 / clean restart / save success

Урок:
[reusable rule]

## Когда делать v2

Делай v2, если:

- идея в целом близко
- композиция или стиль работают
- нужно исправить 1-3 конкретных проблемы
- пользователь оценил результат примерно 6-8/10

## Когда делать v3

Делай v3, если:

- v2 улучшила часть проблем
- осталось 1-2 локальных дефекта
- identity/product/style еще не деградировали

## Когда делать clean restart

Делай clean restart, если:

- после 3-5 правок качество деградирует
- накопились противоречивые инструкции
- лицо, товар или стиль заметно поплыли
- пользователь оценил ниже 5/10
- результат совсем не похож на цель

## Clean restart formula

Using the best current result as the new reference, recreate the scene cleanly. Preserve [what worked]. Fix [main problems]. Remove accumulated artifacts and conflicting instructions. Keep the prompt focused on the final desired image, not on the full history of failed attempts.

## Правило

Каждая новая версия должна отвечать на вопрос: что именно изменилось и зачем.
